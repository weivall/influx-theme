Example Form (With Labels)

When your goal is usability, use form labels. They are less aesthetic and increase the height of your form, but studies have indicated that this approach results in the least amount of user errors. This is because you can see the label and the field at the same time.

Basic Implementation
The entirety of the form is wrapped in <form> tags. Within that each form element is wrapped in div.form-group and paired with <label> for screen-reader purposes.

<form>
  <div class="form-group">
    <label for="example-input-id">Example Input</label>
        <input class="form-control" id="example-input-id" type="text" />
  </div>
</form>

Using the Bootstrap Grid
You can add on the bootstrap grid classes to div.form-group to make the form more interesting and space-efficient. This can be achieved simply by adding the desired grid classes:

<div class="form-group col-sm-6">
  ...
</div>

Example Form (Without Labels)

This type of form is most common in newer web and mobile apps. It instead relies on placeholder text to guide the user, which disappears once they have inputted something. This approach assumes the user is savvy enough to fill out the form even if the input labels are not always present. It can be useful if you are trying to save space, or are going for a more minimalistic aesthetic in your interface.

Implementation
These forms are built the same as the above example, except for a couple differences. Instead of deleting the <label> element you simply add the .sr-only class which blocks it from rendering, but keeps it available for screen reading software. This keeps the form compliant with usability standards. Since there is now no label, you probably want to add a placeholder to the input.

<div class="form-group">
  <label for="example-input-id" class="sr-only" >Example Input</label>
  <input class="form-control" id="example-input-id" type="text" placeholder="Example Placeholder" />
</div>

Effective Placeholders
It may seem obvious to set the placeholder to the same value as the (now obscured) form label, but this is not always true. For example, in an email form one could leave the placeholder as Email Address and call it a day. This can be improved slightly, as the user has no not only recall their email but the appropriate format as well. You can improve the recall time by instead using a placeholder with an example email yourname@website.com. It's a small difference, but it contributes to faster form completion times which in turn improve the user's sentiment about the product. Use examples whenever possible

Example Form (Horizontal Style)

Bootstrap offers a variant on form layouts called "Horizontal Forms". This layout keeps all the labels to the left of their respective inputs for a more aesthetically pleasing layout. This approach generally only works for forms with very few inputs (such as a login form, or lost password) because it takes the eye more time to digest the information. Scanning left and right between label and input is slower than having the label above the input.

Basic Implementation
To use this style of form, all you have to do is add the .form-horizontal class to your form element. The other difference is instead of using grid classes on .form-group you use them on both the <label> and <input> elements. Here's an example:

<div class="form-group">
  <label for="example-input-1" class="col-sm-3">Example Input</label>
  <input id="example-input-1" class="form-control col-sm-9" type="text" />
</div>

Example Form (Inline)

Inline forms are the most minimal, compact form you can create. Hence the name, they are designed for forms small enough to fit on a single line. It is common to combined Form Add-Ons with an inline form for added context (see examples to the right).

Basic Implementation:
Add the .form-inline class to your form element. This does not affect form groups, but does affect their children. Everything inside a form group will now behave like an inline-block element, including the form label. In these examples I have obscured the label using label.sr-only to keep the form cleaner, but you can a keep them if you like.

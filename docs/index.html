Buttons

How to Use

Buttons are to software as links are to web sites. They serve to confirm user input, activate tools, change views, etc. Depending on the context you may want a button of a different size or color. In bootstrap there are two ways to make a button:

1. Anchor Tag for when you want a text link to look and behave like a button

<a class="btn btn-default" role="button" href="#">Button Text</a>

2. Button Tag for when you want an actual button

<button class="btn btn-default">Button Text</button>

Modals

How to Use

Definition from Webopedia

A modal window is any type of window that is a child (secondary window) to a parent window and usurps the parent's control. It is commonly associated with an Internet Web site pop-up window that stays in front of the original window. A user may not press any controls or enter any information on the parent window (the original window which opened the modal) until the modal has been closed. A modal window is commonly used when the author wants to retain the user's focus on the information in the modal as it is impossible for the user to interact with the other windows of the same process.

Modals & UX
Modals are one of the most extreme ways to get a user's attention. Hence the name, modals (or "Modal Dialogs", "Modal Windows") cause a complete shift in the modality of the user. Thus it is best to only use a modal when it feels expected, in the interest of reducing cognitive overhead.

Examples of when to use a Modal

  • User exits current view without saving changes
    This action could have expensive consequences, so it is worth it to hijack the user's attention
  • User deletes a large amount of data
    Also an expensive action, wise to double check before completing.
  • User clicks on a thumbnail
    It is common for thumbnails (or other elements with zoom affordance) to trigger a modal with a larger version of the same thing

Transparency and Context Switching
Switching contexts is super expensive for the user, and some user flows have too much context switching. Modals provide a great way to focus the user on a (usually) small experience without completely divorcing them from their previous task. The transparent background underneath Modals tells the user that they haven't completely left what they were just doing. It's as if the interface is saying "Do this real quick then I'll be out of your way". In effect, modals are micro-pages layered on top of an existing page.

The smaller a Modal is, the less "in your face" it feels. It is best to keep modals as small as possible.

How not to use a Modal
If you aren't sure whether a modal is the appropriate interaction, see if there are other less intrusive means such as a Popover. In most cases, the modal feels heavyhanded when other options are available.

Implementation
See Bootstrap Docs: Javascript - Modals

Form Elements

Example Form (With Labels)

When your goal is usability, use form labels. They are less aesthetic and increase the height of your form, but studies have indicated that this approach results in the least amount of user errors. This is because you can see the label and the field at the same time.

Basic Implementation
The entirety of the form is wrapped in <form> tags. Within that each form element is wrapped in div.form-group and paired with <label> for screen-reader purposes.

<form>
  <div class="form-group">
    <label for="example-input-id">Example Input</label>
        <input class="form-control" id="example-input-id" type="text" />
  </div>
</form>

Using the Bootstrap Grid
You can add on the bootstrap grid classes to div.form-group to make the form more interesting and space-efficient. This can be achieved simply by adding the desired grid classes:

<div class="form-group col-sm-6">
  ...
</div>

Example Form (Without Labels)

This type of form is most common in newer web and mobile apps. It instead relies on placeholder text to guide the user, which disappears once they have inputted something. This approach assumes the user is savvy enough to fill out the form even if the input labels are not always present. It can be useful if you are trying to save space, or are going for a more minimalistic aesthetic in your interface.

Implementation
These forms are built the same as the above example, except for a couple differences. Instead of deleting the <label> element you simply add the .sr-only class which blocks it from rendering, but keeps it available for screen reading software. This keeps the form compliant with usability standards. Since there is now no label, you probably want to add a placeholder to the input.

<div class="form-group">
  <label for="example-input-id" class="sr-only" >Example Input</label>
  <input class="form-control" id="example-input-id" type="text" placeholder="Example Placeholder" />
</div>

Effective Placeholders
It may seem obvious to set the placeholder to the same value as the (now obscured) form label, but this is not always true. For example, in an email form one could leave the placeholder as Email Address and call it a day. This can be improved slightly, as the user has no not only recall their email but the appropriate format as well. You can improve the recall time by instead using a placeholder with an example email yourname@website.com. It's a small difference, but it contributes to faster form completion times which in turn improve the user's sentiment about the product. Use examples whenever possible

Example Form (Horizontal Style)

Bootstrap offers a variant on form layouts called "Horizontal Forms". This layout keeps all the labels to the left of their respective inputs for a more aesthetically pleasing layout. This approach generally only works for forms with very few inputs (such as a login form, or lost password) because it takes the eye more time to digest the information. Scanning left and right between label and input is slower than having the label above the input.

Basic Implementation
To use this style of form, all you have to do is add the .form-horizontal class to your form element. The other difference is instead of using grid classes on .form-group you use them on both the <label> and <input> elements. Here's an example:

<div class="form-group">
  <label for="example-input-1" class="col-sm-3">Example Input</label>
  <input id="example-input-1" class="form-control col-sm-9" type="text" />
</div>

Basic Implementation:

Responsive Tables

The name is somewhat misleading, as the styles are unaffected by screen size. This is for when you have a table with too much stuff in it and it's starting to look cramped. Instead of trying to crush it all in there, "Responsive" Tables enable the table to scroll horizontally.

NOTE:
Table cells inside a Responsive Table do not wrap text. This is partially to encourage overflow (makes it more obvious you can scroll), and to preserve legibility within individual cells. If you are displaying a long string (for example) you may want to consider truncating it as much as possible.

<div class="table-responsive">
  <table class="table table-bordered"> ... </table>

Tooltips

How to use

Tooltips are used to display additional information about a UI element on hover. Tooltips cater to the curious mouse. The benefit of using tooltips is a less cluttered UI.

Implementation:
Tooltips can be added to an element via custom attributes. The content of the Tooltip comes from the title attribute.

<button type="button" class="btn btn-default" data-toggle="tooltip" data-placement="left" title="Tooltip on left">Tooltip on left</button>

Tooltips won't appear until Bootstrap's Tooltips are initialized via:

$(function () {
  $('[data-toggle="tooltip"]').tooltip()
});

Read more about Tooltips on Bootstrap Docs: Javascript - Tooltips

Contextual Alternatives:
In some cases you may want a different context color for a tooltip. Simply add the modifier class to the element that triggers the tooltip.

Primary element.tooltip-primary
Success element.tooltip-success
Warning element.tooltip-warning
Danger element.tooltip-danger
Info element.tooltip-info

Popovers

Default Popover

When you want something kind of like a Tooltip but appears on click instead of hover. Also useful because you can pass in a title and content separately. If you are feeling really fancy, you can inject custom HTML into the Popover.

Common Use Cases

  • Quick preview of shopping cart from navbar
  • Quick preview of a clothing item, doesn't cause user to leave page
  • Asking confirmation of delete without using a modal

Implementation

Popovers are activated via custom HTML attributes applied to an element. In these examples a <button> is used, but you could theoretically use anything.

<button
  type="button"
  class="btn btn-primary"
  data-toggle="popover"
  title="Popover title"
  data-content="Popover content goes here"
>
  Click to toggle popover
</button>

It is possible to use data-content and title without each other for a simpler Popover

Positioning

By adding a third custom HTML attribute you can control where the popover appears relative to the toggle. If no value is specified, the default is "right"

data-placement="top"
data-placement="right"
data-placement="bottom"
data-placement="left"

Limitations

Since the component makes use of custom attributes, you can only pass in plaintext. To inject custom HTML into a popover, follow Boostrap's Popover instructions.

Popovers have a maximum width of 276px

Dismissable Popover

Normally Popovers can only be dismissed by clicking the same element that triggered it. Unless the Popover is intended to be in your face, it is best to make it dismissable. This causes the Popover to dismiss on the next click, regardless of where it is.

Implementation

<button type="button" class="btn btn-primary"
  data-toggle="popover"
  title="Popover title"
  data-content="Popover content goes here"
  data-trigger="focus"
>
  Click to toggle popover
</button>

The look & feel of the Popover can be modified to fit its context. By default Popovers are blue & white. This can be done through yet another custom attribute:

Implementation

  title="Popover title"
  data-content="Popover content goes here"
  data-trigger="focus"
  data-theme="data-explorer"
>
  Click to toggle popover

Progress Bars

Basic Bars

Fairly self explanatory, you've seen these before. The only design consideration is showing the progress as a number or hiding it. Depending on the use of the progress bar, having a label can be helpful or cluttering.

Implementation

<div class="progress">
  <div class="progress-bar" style="width: 60%;">
    <span class="sr-only">60%</span>
  </div>
</div>

.sr-only stands for "Screenreader Only" and hides the element such that only screenreaders can see it. This is for accessibility. If you remove the class, the label text will be visible inside the progress bar.

Collapsible Panels

How to Use

Collapsible panels are a combination of some custom CSS and Bootstrap's built in Collapse javascript. This component is useful when you want to list multiple items in rows, and each item has a too much attached information to show all at once.

Collapsible panels can behave one of two ways:

  1. Default Style - Collapsible panels can be opened / closed separately of each other.
  2. Accordion Style - When only one collapsible panel in a list can be open at one time. Opening a panel closes all other open panels.

Implementation

Since the collapse functionality is built-in, this is pretty straightforward. Collapse works by wrapping an element with a collapsing <div> and assigning it a trigger via custom HTML attributes. In these collapsible panels, div.panel-body gets wrapped.

<div class="panel panel-collapse">
  <div class="panel-heading">
    <h2 class="panel-title">
      <a role="button" data-toggle="collapse" class="collapsed" href="#panel-id-here">
        <span class="caret"></span> Click to Toggle
      </a>
    </h2>
  </div>
  <div class="collapse" id="panel-id-here" style="height: 0px;">
    <div class="panel-body">
      ...
    </div>
  </div>
</div>

Adding style="height: 0px;" sets the collapsible panel to be closed by default

Heatmap Progress Bars

How to Use

For when you want to show a little more urgency when a progress bar is filling up. Useful for Disk Space or Plan Usage.

Implementation
You must use the inverse of your value, since the design shows the value in the negative space. For example, if you wanted to display 20% progress, you would use 80% instead.

<div class="progress progress-heatmap">
  <div class="progress-bar" style="width: 40%;">
    <span>40%</span>
  </div>
</div>

NOTE: Since the modifier class is applied to the parent element insted of the progress bar itself, there is no need for a modifier class on div.progress-bar

Hero

How to use it

Theory

In design speak a "Hero" is the large (sometimes massive) full-width header atop a page. They typically have a visually stunning photo or illustration, sometimes a video, and are the most attention grabbing element on the page. A lot of landing pages have heros with a bold tagline and a call to action button. Heros not only look pretty but tend to convert pretty well also. Heros work best when the content inside is focused, short, and has a clear call to action button. Heros are usually the first element on the page, but this is not a hard rule.

Implementation

In bootstrap, page sections are designated with a div.container. This element has margins on either side, giving it the appearance of being centered. Because Heros are full-width by definition, you must wrap the container you want to be a hero, in the hero div

<div class="hero hero-light">
  <div class="container">
    ...
  </div>
</div>

Hero Types

hero Assumes a light colored background, uses dark text for contrast
hero-light Assumes a dark colored background, uses light text for contrast

Margin / Padding Utilities

How to Use

These are simple utility classes that can be added to an element to avoid writing all new CSS. On the occasion you need to modify the margin or padding in a single instance these are incredibly useful. The classes are named to be self-explanatory and easy to remember.

The flow I like to use these utilities with is:

  • Build the UI using only utility classes and avoiding new CSS as much as possible
  • Review when done, see if any of the utility class CSS should be worked into a new style or not

Some are averse to stacking multiple classes, as it can become hard to scan. I find this not to be a problem, as it allows one to maintain flow without having to swap between markup and styles, resulting in more productivity.

Code Highlighting

How to Use

There are many code highlighting JS libraries out there, for example we use Prism.js to handle code highlighting in our Developer Docs. There is no built-in code highlighting within Bootstrap, so I have created a set of highlighters for manual use. In the future there will be a code highlighting feature built into the theme.

Concept
In HTML there are a variety of tags specifically for formatting text. Most websites and applications no longer use these tags as the same effect can be achieved with CSS more cleanly. I have repurposed these tags to function as code highlighters, scoped within code samples.

Taking into consideration the possibility that a external code highlighter might be used in conjunction with this theme, I have put a failsafe in play:

For Code Highlighting to take effect, just add a class to your sample,

<pre class="rainbow-highlighter">
  <code></code>
</pre>

<code class="rainbow-highlighter"> </code>

Then use these tags to apply highlights:

Flex Box Utilities

These utility classes are designed for use on any element. They are modular and are helful when you want to use a flex-box based layout but don't want to write additional custom classes


Class What it does
.u-flex When used on an element, sets display: flex;. Usually placed on the parent element in which you want to layout the children with flex box.
Align Items (ai) Used on the parent element to vertically align children
.u-ai-center Aligns children to their vertical centers
.u-ai-stretch Stretches children vertically to match the height of the parent
.u-ai-baseline Aligns children vertically to their textual baselines
.u-ai-flex-start Aligns children to the start of the flex object (the top of the parent)
.u-ai-flex-end Aligns children to the end of the flex object (the bottom of the parent)
Justify Content (jc) Used on the parent to control horizontal distribution of children
.u-jc-flex-start Aligns children to the beginning of the flex object (the left edge of the parent)
.u-jc-flex-end Aligns children to the end of the flex object (the right edge of the parent)
.u-jc-center Aligns children to the horizontal center of the flex object
.u-jc-space-between Puts as much space between each child as possible
.u-jc-space-around Puts as much space around each child as possible
Align Content (ac) Used on the parent to control positioning of the children when there are multiple rows of children
.u-ac-flex-start Aligns each row of children to the start of the flex object (the top of the parent)
.u-ac-flex-end Aligns each row of children to the end of the flex object (the bottom of the parent)
.u-ac-center Aligns all rows of children to the parent's vertical center
.u-ac-stretch Stretches all rows of children such that they fill the height of the parent
.u-ac-space-between Maximizes space between each each row of children
.u-ac-space-around Maximizes space around each each row of children
